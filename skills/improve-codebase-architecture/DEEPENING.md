# Deepening Opportunities

Look for these patterns to find deepening opportunities:

1.  **Pass-through modules**: Modules that just wrap another module without adding significant value.
2.  **Tangled dependencies**: Modules that are tightly coupled and leak implementation details.
3.  **Untestable logic**: Logic that is hard to test because it's buried deep inside a shallow module.
4.  **Inconsistent terminology**: Modules that use different names for the same domain concept.
